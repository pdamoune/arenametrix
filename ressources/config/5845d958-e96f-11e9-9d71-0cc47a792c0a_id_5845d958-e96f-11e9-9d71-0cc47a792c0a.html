Test Python
I- Contexte
Le but de ce test est de réaliser une API permettant d’importer, stocker, et extraire des données
provenant d’un fichier CSV.
Le fichier de données initial correspond à une liste de réservations contenant les informations sur
l'acheteur, l'événement et la tarification.
II- Résultat attendu
L'application doit comprendre :
●
Une url d'import de fichier CSV
●
Une url récapitulative des données en base
III- Exigences fonctionnelles
1. Je peux importer les données à partir d’un fichier CSV aux colonnes définies.
L'url d’import me permet d’uploader un fichier CSV dont on considère que la forme (nom et ordre
des colonnes) sera toujours la même. A l’issue de l’import, les données contenues doivent être
sauvegardées dans une base relationnelle.
Hors périmètre :
Le choix du SGBDR et du framework sont libre.
●
Certaines spécificités du domaine métier peuvent être interprétées de différentes façons lors de
●
la modélisation des données, en l’absence de connaissances de l’existant. Tant que l’ensemble
reste logique, il n’en sera pas tenu rigueur.
2. Je peux extraire les indicateurs clés représentant les publics ainsi que la performance globale.
Sur une seconde url, je peux extraire l'ensemble des indicateurs suivants :
●
le nombre de réservations
●
le nombre d’acheteurs uniques
●
l’âge moyen des acheteurs
●
le prix moyen par représentation
●
le prix moyen par client
Hors périmètre :
L'arrondi ou unité des valeurs ne sont pas utiles, ces considérations seront prise en compte par le
●
client qui appel l'url.
